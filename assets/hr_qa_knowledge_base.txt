# 🔹 TECH FOLLOW-UP (IMPORTANT)

**Q: What happens inside a RAG system?**
A:

**Point:**
RAG combines retrieval with generation to produce context-aware answers.

**Reason:**
LLMs alone don’t have updated or personal data.

**Example:**
In my system:

* Documents → converted into embeddings
* Stored in vector DB (ChromaDB)
* Query → converted into embedding
* Similar chunks retrieved
* LLM generates answer using that context

**Point:**
So RAG ensures answers are both accurate and relevant.

**Q: Difference between RAG and Agent?**
A:

**Point:**
RAG retrieves and answers, while an agent can decide actions.

**Reason:**
RAG is passive, agent is active.

**Example:**

* RAG → fetch + answer
* Agent → think → choose tool → act → answer

**Point:**
So agent systems are more flexible and powerful.
